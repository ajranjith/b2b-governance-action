CLI Setup Guide

Step-by-step commands to install and configure gres-b2b

1

Install gres-b2b CLI

Run the one-line installer for your platform:

curl -fsSL https://ajranjith.github.io/b2b-governance-action/install.sh | bash

Installs to ~/.local/bin (or /usr/local/bin with sudo).

irm https://ajranjith.github.io/b2b-governance-action/install.ps1 | iex

Installs to %LOCALAPPDATA%\gres-b2b. No admin required.

2

Verify Installation

Open a new terminal (so PATH is updated) and run:

gres-b2b --version
Expected output 
gres-b2b version 1.x.x
3

Configure MCP for Your AI Agent

Add gres-b2b as an MCP server in your AI agent's configuration:

Edit %APPDATA%\Claude\claude_desktop_config.json (Windows) or ~/Library/Application Support/Claude/claude_desktop_config.json (macOS):

{
  "mcpServers": {
    "gres-b2b": {
      "command": "gres-b2b",
      "args": ["mcp", "serve"]
    }
  }
}

Restart Claude Desktop after saving.

Open Cursor Settings → Features → MCP Servers → Add Server:

Name: gres-b2b
Command: gres-b2b mcp serve

Install the MCP extension, then add to your settings.json:

{
  "mcp.servers": {
    "gres-b2b": {
      "command": "gres-b2b",
      "args": ["mcp", "serve"]
    }
  }
}
4

Test MCP Connection

Verify that MCP is working correctly with your AI agent:

gres-b2b mcp selftest
Expected output 
MCP selftest passed
Protocol: stdio
Handshake: OK
Troubleshooting

If selftest fails, ensure your AI agent is running and MCP is enabled in its settings.

5

Check Prerequisites

Run the doctor command to verify all prerequisites are met:

gres-b2b doctor
Expected output 
Doctor check passed
All prerequisites OK
6

Run Your First Scan

Navigate to your project folder and run a governance scan:

cd /path/to/your/project
gres-b2b scan

This generates your governance report at .b2b/report.html.

You're all set! 
Open .b2b/report.html to view your RED/AMBER/GREEN dashboard
